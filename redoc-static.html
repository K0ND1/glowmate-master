GlowMate API (1.2.0)

Download OpenAPI specification:

API for the GlowMate application, which helps users take care of their skin. It allows for product scanning, ingredient analysis, rating, and personalization of skincare routines using AI.

Performance Notes:

  • Complex queries with multiple filters may take 2-5 seconds
  • Consider using fewer filters or more specific search terms for faster results
  • Popular filter combinations are cached for optimal performance

Rate Limits:

  • Authenticated users: 100 requests/minute
  • IP-based limit: 1000 requests/minute

Database Requirements:

  • Full-text search index on ingredients array
  • Compound index on (tags, brand, price, averageRating)
  • B-tree indexes on price and averageRating fields

Products

Scan, search, rate, and add cosmetic products.

Search and filter products

Returns a paginated list of products with advanced filtering capabilities.

Ingredient Filtering:

  • Use includeIngredients to find products containing ALL specified ingredients (AND logic)
  • Use excludeIngredients to exclude products with ANY of the specified ingredients
  • Ingredient matching is case-insensitive and normalized
  • Maximum 10 ingredients per filter to ensure performance

Tag Filtering:

  • Multiple tags use OR logic (product matches if it has ANY of the specified tags)
  • Use tagsLogic=all to require ALL tags (AND logic)

Examples:

  • Find moisturizers with Niacinamide: ?includeIngredients=Niacinamide&tags=moisturizing
  • Find products without Alcohol or Fragrance: ?excludeIngredients=Alcohol,Fragrance
  • Complex filter: ?includeIngredients=Niacinamide,Hyaluronic Acid&excludeIngredients=Fragrance&minRating=4.0&minPrice=10&maxPrice=50

Performance Tips:

  • Use specific search terms in q parameter to reduce result set
  • Limit the number of active filters for faster response times
  • Popular filter combinations are automatically cached
Authorizations:
bearerAuth
query Parameters
q
string <= 100 characters
Example: q=moisturizer

Text to search in the product name or brand. Maximum 100 characters.

includeIngredients
Array of strings [ 1 .. 10 ] items [ items <= 100 characters ]
Example: includeIngredients=Niacinamide,Hyaluronic Acid

Comma-separated list of ingredients that products MUST contain (AND logic). Case-insensitive. Maximum 10 ingredients.

excludeIngredients
Array of strings [ 1 .. 10 ] items [ items <= 100 characters ]
Example: excludeIngredients=Alcohol,Fragrance

Comma-separated list of ingredients that products MUST NOT contain. Case-insensitive. Maximum 10 ingredients.

tags
Array of strings [ 1 .. 5 ] items
Items Enum: "moisturizing" "anti-aging" "acne-prone" "brightening" "soothing" "exfoliating" "sunscreen" "cleansing" "serum" "night-cream" "day-cream"
Example: tags=moisturizing,anti-aging

Comma-separated list of product tags/categories. Default logic: OR (any tag matches). Maximum 5 tags.

tagsLogic
string
Default: "any"
Enum: "any" "all"

Logic for combining multiple tags. 'any' = OR logic (default), 'all' = AND logic.

brands
Array of strings [ 1 .. 10 ] items [ items <= 100 characters ]
Example: brands=CeraVe,La Roche-Posay

Comma-separated list of brands to filter by. Maximum 10 brands.

minPrice
number <float> [ 0 .. 100000 ]
Example: minPrice=10

Minimum price (inclusive). Must be less than maxPrice if both are specified.

maxPrice
number <float> [ 0 .. 100000 ]
Example: maxPrice=50

Maximum price (inclusive). Must be greater than minPrice if both are specified.

minRating
number <float> [ 0 .. 5 ]
Example: minRating=4

Minimum average rating (inclusive). Range: 0-5.

limit
integer [ 1 .. 100 ]
Default: 20

Number of results per page.

offset
integer >= 0
Default: 0

Number of results to skip (for pagination).

sortBy
string
Default: "relevance"
Enum: "name" "brand" "rating" "price" "newest" "relevance"

Field to sort the results by. 'relevance' is only available when using text search (q parameter).

sortOrder
string
Default: "asc"
Enum: "asc" "desc"

Sort order: ascending or descending. Default is 'desc' for 'rating', 'newest', and 'relevance'.
